Basic template

How to start a page with G.R.I.D ?

Use the HTML below to begin working with a minimal G.R.I.D document.

<!DOCTYPE html>
<html>
    <head>
        <meta http-equiv="content-type" content="text/html; charset=utf-8" />
        <meta name="viewport" content="width=device-width; initial-scale=1.0" />
        <!-- The above meta tags *must* come first in the head; any other head content must come *after* these tags -->

        <title>G.R.I.D minimal template</title>

        <!-- G.R.I.D CSS -->
        <link rel="stylesheet" type="text/css" href="css/g.r.i.d.min.css" />

        <!-- You can add here some Javascript tweaks like respond.js and html5shiv.js for IE8 support of HTML5 and media queries -->
            <!--[if lt IE 9]>
            <script src="https://oss.maxcdn.com/html5shiv/3.7.2/html5shiv.min.js"></script>
            <script src="https://oss.maxcdn.com/respond/1.4.2/respond.min.js"></script>
        <![endif]-->

    </head>

    <body>
        <!-- This div is very important ! All your G.R.I.D documents must have a div with an id "page" which englobe all your body contents -->
        <div id="page">
            <!-- You can use it for your header -->
            <headerr id="header-wrapper">
                <!-- This div is also important. Before insert any content you have to create this div -->
                <div class="container">
                    <h1>Welcome !</h1>
                </div>
            </header>

            <!-- You can use it for your contents -->
            <div id="main-wrapper">
                <div class="container">
                </div>
            </div>

            <!-- You can use it for your footer -->
            <footer id="footer-wrapper">
                <div class="container">
                </div>
            </footer>
        </div>

        <!-- jQuery (necessary for G.R.I.D's JavaScript plugins) -->
        <script type="text/javascript" src="https://ajax.googleapis.com/ajax/libs/jquery/1.11.3/jquery.min.js"></script>
        <!-- Include all plugins -->
        <script type="text/javascript" src="js/g.r.i.d.min.js"></script>
    </body>
</html>

This template structure is highly recommended to start a G.R.I.D document ! All example templates use this structure.

Using G.R.I.D CSS

Use the power of G.R.I.D !

Please, see G.R.I.D CSS demo first: G.R.I.D CSS Demo

Open a G.R.I.D

When you want to open a new G.R.I.D, you have to create an HTML element with a class grid

<!-- Creating a new G.R.I.D with a div -->
<div class="grid">…</div>

<!-- Creating a new G.R.I.D with an ul -->
<ul class="grid">…</ul>

<!-- Creating a new G.R.I.D with an ol -->
<ol class="grid">…</ol>

<!-- Creating a new G.R.I.D with a section -->
<section class="grid">…</section>

<!-- … You can create a G.R.I.D with any HTML element … -->

G.R.I.D have 5 others variants (See the G.R.I.D CSS demo to learn more)

Insert and manage G.R.I.D elements

After you have opened a new G.R.I.D, you have to insert elements ! G.R.I.D comes with 12 levels of elements and each element have a class, so we have :

  • The class g12 to insert an element which will take the 100% of the entire space of the G.R.I.D
  • The class g11 to insert an element which will take the 91.6666666667% of the entire space of the G.R.I.D
  • The class g10 to insert an element which will take the 83.3333333333% of the entire space of the G.R.I.D
  • The class g9 to insert an element which will take the 75% of the entire space of the G.R.I.D
  • The class g8 to insert an element which will take the 66.6666666667% of the entire space of the G.R.I.D
  • The class g7 to insert an element which will take the 58.3333333333% of the entire space of the G.R.I.D
  • The class g6 to insert an element which will take the 50% of the entire space of the G.R.I.D
  • The class g5 to insert an element which will take the 41.6666666667% of the entire space of the G.R.I.D
  • The class g4 to insert an element which will take the 33.3333333333% of the entire space of the G.R.I.D
  • The class g3 to insert an element which will take the 25% of the entire space of the G.R.I.D
  • The class g2 to insert an element which will take the 16.6666666667% of the entire space of the G.R.I.D
  • The class g1 to insert an element which will take the 8.3333333333% of the entire space of the G.R.I.D
Note that all of these elements will take the 100% of the entire space of the G.R.I.D in a phone view !

You can combine all these elements in G.R.I.D at the condition that the sum of widths equals 100%.

<!-- Adding an element of level 12 in the G.R.I.D. This element will take the 100% of the G.R.I.D -->
<div class="grid">
    <div class="g12">
        <p>Some text</p>
    </div>
</div>

<!-- Adding two elements of level 6 in the G.R.I.D. These elements will take the 50% of the G.R.I.D (50% + 50% = 100%) -->
<div class="grid">
    <div class="g6">
        <p>Some text</p>
    </div>
    <div class="g6">
        <img src="foo/bar.png" alt="Some text" />
    </div>
</div>

<!-- Adding three elements, one of level 6 and two of level 3 in the G.R.I.D. Sure that the sum of the widths of these elements will equals 100% ;-)   -->
<div class="grid">
    <div class="g3">
        <div class="sidebar left">
            <div class="calendar">
                <!-- You can put a calendar here -->
            </div>
        </div>
    </div>
    <div class="g6">
        <p>Some text</p>
    </div>
    <div class="g3">
        <div class="sidebar right">
            <div class="widget">
                <!-- You can put a widget here -->
            </div>
        </div>
    </div>
</div>

<!-- … You can do all matches you want ! … -->
When a G.R.I.D is full (when its elements takes all the space), you have to open a new one. Don't insert a new element in a full G.R.I.D !.

It's highly recommended to insert your contents after having insert a G.R.I.D element, not before, with, or directly in the G.R.I.D.

<!-- Bad use of the G.R.I.D: No G.R.I.D elements are inserted in the G.R.I.D before inserting contents -->
<div class="grid">
    <section class="box">
        <p>Some text</p>
    </section>
</div>

<!-- Another bad use of the G.R.I.D: The content is inserted with the G.R.I.D element -->
<div class="grid">
    <section class="g12 box">
        <p>Some text</p>
    </section>
</div>

<!-- Good use of the G.R.I.D -->
<div class="grid">
    <div class="g12">
        <section class="box">
            <p>Some text</p>
        </section>
    </div>
</div>

You can push right some G.R.I.D elements to arrange the aspect. Use it only if your G.R.I.D is not full. To push an element, you have to add the class which represent the level of pushing :

  • The class -g11 to push an element 91.6666666667% to the right of the G.R.I.D
  • The class -g10 to push an element 83.3333333333% to the right of the G.R.I.D
  • The class -g9 to push an element 75% to the right of the G.R.I.D
  • The class -g8 to push an element 66.6666666667% to the right of the G.R.I.D
  • The class -g7 to push an element 58.3333333333% to the right of the G.R.I.D
  • The class -g6 to push an element 50% to the right of the G.R.I.D
  • The class -g5 to push an element 41.6666666667% to the right of the G.R.I.D
  • The class -g4 to push an element 33.3333333333% to the right of the G.R.I.D
  • The class -g3 to push an element 25% to the right of the G.R.I.D
  • The class -g2 to push an element 16.6666666667% to the right of the G.R.I.D
  • The class -g1 to push an element 8.3333333333% to the right of the G.R.I.D
Note that all of these pushing levels will not work on a phone view !
<!-- Inserting two elements, and push the last one to the right -->
<div class="grid">
    <div class="g4">
        <p>Some text</p>
    </div>
    <div class="g4 -g4">
        <p>Another text</p>
    </div>
</div>
<!-- Note that the level of pushing -g4 is the maximum possible, because the sum of all these levels equals 12 (4 + 4 + 4 = 12) -->

You can also force the G.R.I.D to respect elements's widths on all viewports. You only have to add the class force on the G.R.I.D.

<!-- Force to show these two elements side by side on a phone view -->
<div class="grid force">
    <div class="g6">
        <p>Some text</p>
    </div>
    <div class="g6">
        <p>Another text</p>
    </div>
</div>

The class force will also maintain the width of all pushing levels.

<!-- Center this element on all viewports -->
<div class="grid force">
    <div class="g6 -g3">
        <p>Centered element !</p>
    </div>
</div>

If you want to change the size of an element according to the viewport, you can use modifiers :

  • (pc) to set the element's width only on pc/desktop
  • (tablet) to set the element's width only on tablets
  • (phone) to set the element's width only on phones
<!-- Change the width of an element according to the viewport -->
<div class="grid">
    <div class="g12 g6(phone)">
        <p>Some text</p>
    </div>
</div>

In this example, the second div will take the 100% of the entire space of the G.R.I.D in PC and tablet viewports, and will take the 50% of the entire space of the G.R.I.D only on phone viewports.

You can also hide some elements according to the viewport, by using the class hide with one of the modifiers.

<!-- An example of the use of modifiers with hide -->
<div class="grid">
    <div class="g6 hide(tablet) hide(phone)">
        <p>This text will only be showed on PCs</p>
    </div>
    <div class="g6 hide(pc) hide(phone)">
        <p>This text will only be showed on tablets</p>
    </div>
    <div class="g6 hide(pc) hide(tablet)">
        <p>This text will only be showed on phones</p>
    </div>
    <div class="g6">
        <p>This text will be showed on all viewports</p>
    </div>
</div>

Disable responsiveness

G.R.I.D will automatically adapts your website for various screen sizes. But there are some steps to disable this capability :

  1. Ommit the viewport <meta> mentioned in the basic template.
  2. Add the class non-responsive to the <body> of your document.

Once the responsiveness disabled :

  1. All G.R.I.D elements will maintain their width on all viewports.
  2. The use of force is ignored.
  3. The use of modifiers is ignored.
  4. The use of hide is ignored.
